Everything in javascript happens inside the execution context

It is like a big box and it has 2 components:-

1.memory which has all the variables and functions. This memory component is also called as Variable enviroment

2. Code component it is also known as Thread of execution It is like a thread in which code is executed line by line

Javascript is a synchronous single-threaded language

Single threaded means it can execute one command at a time and synchronous single threaded means one at a time in a specific order

Execution is done in 2 phases:

Phase1- Memory creation- allocated memory to var and functions. In case of var it stored keyword undefined and in functions it stores the whole function code

Phase 2- Code Execution Phase all the calculations happens in this phase

As soon we are done with function execution, the whole execution context is deleted.

It manages everything in the Call Stack It has GEC and when we invoke a func , it store that func EC and when we are done with the func then the EC of that func is removed from the stack. This is the way how things are done.
